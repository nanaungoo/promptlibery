# 📘 AI Assistant Protocol & IDE Configuration Guide

ဒီလမ်းညွှန်ချက်ဟာ **Vibe Coding** လုပ်နေစဉ်မှာ AI ကို "အတွင်းရေးမှူး" နဲ့ "လက်ထောက်ဆရာ" တစ်ယောက်လို အသုံးချပြီး ကျောင်းသားတွေအတွက် သင်ယူမှုမှတ်တမ်း (Learning Journal) တွေကို အလိုအလျောက် ထုတ်ပေးဖို့ ရည်ရွယ်ပါတယ်။

---

## 1. GEMINI.md (The Master Protocol)

ဒါဟာ သင့် project ရဲ့ **"Single Source of Truth"** ဖြစ်ပါတယ်။ AI က ဘယ်လိုပြုမူရမလဲဆိုတဲ့ စည်းကမ်းတွေနဲ့ Project ရဲ့ အကြောင်းအရာတွေကို ဒီထဲမှာ သိမ်းဆည်းပါတယ်။

```markdown
> **SYSTEM ALERT:** If this is the start of a session, do not wait for user instructions. Immediately update the status of `TODO.md` and check if `LEARNING_JOURNAL.md` needs backfilling.

# 🤖 AI ASSISTANT UNIVERSAL PROTOCOL (V1.0)

## 1. Interaction & Logging Rules
- **Communication:** ALWAYS use **English** for professional dialogue, chat, and CLI interactions.
- **Journaling:** ALWAYS use **Myanmar** when writing to `LEARNING_JOURNAL.md`.
- **Primary Objective:** Keep the developer in the "flow state" (vibe coding) while you handle the documentation and educational recording.

## 2. Live Logging Protocol (Educational)
Maintain `LEARNING_JOURNAL.md` after every successful task or fix.
- **Content (Myanmar):** Task Name, User Intent, AI Reasoning (the "Why"), Debug Log (if any), Implementation Details, Efficiency/Performance note, and Terminal Commands.
- **Formatting:** Use `---` separators and clear headers.

## 3. Session Handoff & Continuity
- **End Session:** Update `TODO.md` with "✅ Completed", "🚧 Next Steps", and "🧠 Context Memory".
- **Resume Session:** Re-read this protocol, `TODO.md`, and `LEARNING_JOURNAL.md` before starting.

---

# 📂 CURRENT PROJECT CONTEXT: [Project Name]

## Project Overview
- **Purpose:** [Explain what you are building]
- **Environment:** [e.g., Termux, Linux, Android]
- **Core Tech:** [e.g., Rust, Python, Meta MMS-TTS]

## Development Conventions
- **Minimalism:** Minimal dependencies for performance.
- **Efficiency:** Use Lazy Imports/Memory Optimization.
- **Standards:** All I/O uses UTF-8.

```

---

## 2. IDE-Specific Rule Files

သက်ဆိုင်ရာ IDE အလိုက် အောက်ပါဖိုင်များကို Project Root Folder ထဲမှာ ဖန်တီးထည့်သွင်းပါ။

### A. Cursor IDE (`.cursorrules`)

```text
Always adhere to the "AI ASSISTANT UNIVERSAL PROTOCOL" defined in GEMINI.md.
- Chat: English Only.
- Journaling: Myanmar Only (in LEARNING_JOURNAL.md).
- Prioritize: "Vibe Coding" flow by handling documentation overhead.

```

### B. Zed AI (`.zed/settings.json`)

```json
{
  "assistant": {
    "version": "2",
    "custom_instructions": "Follow the rules in GEMINI.md. Use English for chat and Myanmar for LEARNING_JOURNAL.md entries. Maintain TODO.md for context."
  }
}

```

### C. Windsurf (`.windsurfrules`)

```text
Role: Senior Mentor.
Instruction Source: GEMINI.md.
Language: English for communication, Myanmar for LEARNING_JOURNAL.md.

```

### D. Aider CLI (`.aider.instructions.md`)

```markdown
# Aider Protocol
- Refer to `GEMINI.md` for project context.
- Commit Messages: English.
- Learning Journal Entries: Myanmar.
- Auto-update `TODO.md` and `LEARNING_JOURNAL.md` after each success.

```

### E. Neovim (Avante/CodeCompanion)

```lua
-- Add to your AI system prompt config
[[ Follow GEMINI.md. English for dialogue, Myanmar for learning logs. Handle TODOs automatically. ]]

```

## 3. Workflow Commands (AI အား ခိုင်းစေရန်)

စီမံကိန်းစတင်တဲ့အခါ ဒါမှမဟုတ် တစ်နေ့တာ အလုပ်စတဲ့အခါ AI ကို အောက်ပါအတိုင်း ပြောနိုင်ပါတယ်။

| အခြေအနေ | AI ကို ပြောရမည့် စကားလုံး |
| --- | --- |
| **ပထမဆုံးစတင်ခြင်း** | "Read GEMINI.md. Initialize `TODO.md` and backfill `LEARNING_JOURNAL.md`." |
| **အလုပ်ပြန်စခြင်း** | "**Resume**" (ဒါမှမဟုတ် "Resume session per protocol") |
| **နားတော့မည့်အခါ** | "**Stop for now.** Update the handoff notes in TODO.md." |
| **AI လမ်းချော်သွားလျှင်** | "Follow the GEMINI.md protocol strictly." |

## 4. Universal Technical Guardrails (နည်းပညာဆိုင်ရာ စည်းကမ်းများ)

AI ကို Code ရေးခိုင်းတဲ့အခါ အောက်ပါအချက်တွေကို အမြဲသတိပေးဖို့ GEMINI.md ထဲမှာ ထည့်ထားပါတယ်။

* **Minimalism:** လိုအပ်တာထက် ပိုတဲ့ dependencies တွေ မသုံးဖို့။
* **Performance:** RAM နဲ့ CPU သက်သာအောင် ရေးဖို့ (အထူးသဖြင့် Termux/Mobile အတွက်)။
* **Lazy Imports:** Python မှာဆိုရင် Library အကြီးစားတွေကို function ထဲမှာမှ ခေါ်သုံးဖို့။
* **Preservation:** အလုပ်လုပ်နေတဲ့ code တွေကို မလိုအပ်ဘဲ Refactor (ပြန်ပြင်) မလုပ်ဖို့။

---

ဟုတ်ကဲ့ပါ၊ တကယ့် Power User တွေနဲ့ Open-source ချစ်သူတွေ အသုံးများတဲ့ **OpenHands (OpenDevin)**၊ **Continue (Codex)**၊ **Aider** နဲ့ **Open Interpreter** တို့အတွက် လိုအပ်တဲ့ Rule တွေကိုပါ အားလုံးပေါင်းပြီး တစ်ခါတည်း သိမ်းထားလို့ရအောင် Markdown format နဲ့ စုစည်းပေးလိုက်ပါတယ်။

ဒါကို `VIBE_CODING_MASTER_GUIDE.md` အနေနဲ့ သိမ်းထားလိုက်ရင် ဘယ် AI Tool ကိုပဲသုံးသုံး အဆင်ပြေသွားပါပြီ။
